Reports Screen

This is the screen for creating custom reports.
Reports have two parts to them:

  • The name of the report.
  • The rules that specify what properties a report should contain
    based on a property's County, Municipality, Zip Code, Body of Water,
    Condo Name, and/or Category.

Naming/Adding a Report

Before any rules can be added to a report, the report must be
created itself. To do this.

  1. Click on the text box that says "Report:" to the left of it.
  2. Enter a name for the report here and click the "Add" button below
    the textbox. This will be what the report is called when it is
    generated as a file. For example, if you named a file "Kenosha",
    then something like "KenoshaQ1.csv" will be the file name. Note
    that Q1 stands for Quarter 1; however, it might Q2, Q3, or Q4
    depending on what quarter you are generating the report for.
  3. After clicking the "Add" button, the report's name will appear
    in the list directly above the textbox. The report has been added.

Adding Rules to a Report

Now that the report has been added and named, we can start adding rules
to it. But before that, rules should be explained a little further.
Each rule in a report essentially describes the contents of the csv report
file. Each rule has two parts: a "where" part and an "equals" part.
The "where" part lets the user select a specific property of an entry (County,
Municipality, Zip Code, Body of Water, Condo Name, and Category). The "equals"
part tells the program to select only the entries with the selected
value. Examples are given under "Example Reports" in order to illustrate
how rules can be used to create reports.
To add a rule:

  1. Under the "Reports:" label (the list that you just added a report to)
    Click on the report that you want to add rules to. If there are any
    rules already added to the report, then they will appear in the
    list labeled "Rules in effect:" right below the Reports list.
  2. Now that the report is highlighted, click on the dropdown box labeled
    "Where". Once clicked, the dropdown menu will show six possible options:
    County, Municipality, Zip Code, Body of Water, Condo Name, and Category.
    Click on the desired option.
  3. Once you click on one of the six options, you will notice that the dropdown
    menu to the right (the "equals" dropdown menu) has been populated with options.
    These are all the values that the program found in the imported CSV file
    for the respective "where" option chosen.
  4. Select the desired item from the "equals" dropdown menu. Once selected,
    click on the "Add" button directly below the dropdown menu. You will now
    see that the rule has been added to the list labeled "Rules in effect:"
    Now, when you click the "Generate Reports" button on the main screen,
    the report that you just created here will be generated with your added rules
NOTE: If no rules are specified for a report, then every property will be included in the report.

Example Reports

Generate a report called "Kenosha-Single-Family-Condo" which contains all of the single-family
and condo properties sold in Kenosha county:
Report name: "Kenosha-Single-Family"
Rules:
Where Category = Condominium
where Category = Single-Family
Where County = Kenosha

Generate a report called "Bloomfield-Brighton-Bristol" which contains all of the
properties sold in Bloomfield, Brighton, and Bristol municipalities:
Report name: "Bloomfield-Brighton-Bristol"
Rules:
Where Municipality = Bloomfield
Where Municipality = Brighton
Where Municipality = Bristol

Generate a report named "Lake Geneva" which contains all of the properties
sold on Lake Geneva:
Report name: ""
Rules:
Where Body of Water = Lake Geneva

Deleting a Report

If a report is deleted, then all of the rules contained within that report are
also deleted. If you would rather delete some of the rules in a report rather than
the entire report, see "Deleting a Rule from a Report" below.
  1. In the "Reports:" list, click on the report you wish to delete.
  2. Once you have highlighted the report, click on the "delete" button
    beneath the "Report:" textfield.

Deleting a Rule from a Report

If you would like to keep the report itself, but delete some of the rules
it contains, then follow these steps:
  1. In the "Reports:" list, select the report you would like to delete the
    rule from.
  2. Once the report has been highlighted, the list labeled "Rules in effect:"
    will be populated with all of the rules inside the report.
  3. In the "Rules in effect:" list, click the rule that you wish to delete.
  4. Once the rule is highlighted, click the delete button beneath the
    "Rules in effect:" list.
Note: Multiple rules can be deleted at one time by selecting multiple rules (Using CNTRL or Shift)
and then clicking delete.

Loading a .rule File

To load a file click the browse button the right or enter the
absolute pathname into the text field below the "Rule file:"
label. After a valid absolute pathname is in the text field, you
can press the "Load" button to load all of the reports into memory.
The reports will be loaded into the list under "Reports:" while the
rules will be loaded into the list under the "Rules in effect"label.
These rules will now be applied when reports are generated.

Saving a .rule File

To load a file click the browse button the right or enter the
absolute pathname into the text field below the "Rule file:"
label. After a valid absolute pathname is in the text field, you
can press the "Save" button to save all of the current Rules in
effect to the given file name. Next time the program is used, you
can load the new .rule file instead of entering all of the reports
and rules manually.

Reports List

This is the list of all of the reports that will be generated.
Click on any of the reports to highlight it and see all of the rules
that it contains. The rules inside the report will populate the the
Rules in Effect List.

Rules in Effect List

This is the list of all of the rules that are currently in effect
for the currently highlighted report and will be used to create the
appropriate files when the reports are generated.
